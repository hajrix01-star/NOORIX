# الأداء والبيانات الكبيرة — Noorix

مرجع لاستراتيجية الكاش المركزي والتعامل مع البيانات الكبيرة لتجنب البطء.

---

## 1. الكاش والبيانات المرجعية

- **React Query** هو المعيار المعتمد:
  - **البيانات المرجعية**: قوائم الموردين، الخزائن، التصنيفات — تُجلب عبر `useQuery` مع `queryKey` موحد (مثل `['suppliers', companyId]`).
  - **الإحصائيات والتجميعات**: لوحة التحكم، أرصدة الخزائن — تُجلب عبر `useQuery` مع مفاتيح مناسبة.
- **الإبطال**: بعد أي mutation مالي استدعِ `invalidateOnFinancialMutation(queryClient)` من `utils/queryInvalidation.js` — يبطل تلقائياً: invoices, vaults, sales-summaries, ledger, reports. يضمن تحديث التقارير والقوائم مباشرة دون تحديث يدوي.
- **الهوكات**: `useSales`, `useInvoices`, `useSuppliers`, `useVaults`, `useCategories` — كلها تستخدم React Query وتُبطل الكاش في `onSuccess`.

---

## 2. كيف نتعامل مع البيانات الكبيرة؟ (لا بطء)

- **الترقيم (Pagination)**:
  - قوائم الفواتير، الحركات، الموظفين، الموردين: **لا تحميل كل الصفوف دفعة واحدة**. طلب صفحة بحجم ثابت (مثلاً 50 أو 100 صف) مع `page` و `pageSize`؛ الـ Backend يرجع `items` و `total`.
  - الواجهة: أزرار أو infinite scroll للصفحة التالية؛ المستخدم يرى جزءاً صغيراً في كل مرة فلا بطء في التحميل أو الرسم.

- **التصفية من الخادم (Server-side filtering)**:
  - البحث والفلترة (تاريخ، مورد، شركة) يجب أن تُنفَّذ في الـ Backend؛ النتائج المرجعة هي الصفحة الحالية فقط. تجنب جلب كل البيانات ثم تصفيتها في المتصفح.

- **الجدول الافتراضي (Virtualization)**:
  - عند عرض **آلاف الصفوف** في صفحة واحدة (مثلاً تقرير تفصيلي): استخدام **جدول افتراضي** (مثل `react-window` أو `@tanstack/react-virtual`) لرسم **فقط الصفوف المرئية** في الشاشة؛ الباقي غير مرسوم في DOM فيقل استهلاك الذاكرة والوقت ولا يحدث تجمد.

- **التصدير (Excel/PDF)**:
  - التصدير يُنفَّذ **من الخادم** عند وجود بيانات ضخمة (مثلاً آلاف الحركات)، أو على دفعات من الواجهة مع **Dynamic Import** لمكتبات XLSX/PDF عند الضغط على "تصدير" فقط حتى لا تثقل الحزمة الأولى.

- **الخلاصة**: مع **ترقيم + تصفية من الخادم + كاش للإحصائيات والمرجعيات + virtualization للجداول الكبيرة عند الحاجة** لن يكون هناك بطء غير مقبول حتى مع بيانات كبيرة.

---

## 3. ملخص التطبيق في المشروع

- **الكاش**: React Query (`@tanstack/react-query`) — `useQuery` للجلب، `queryClient.invalidateQueries` للإبطال بعد mutations. الهوكات في `src/hooks/` (useSales, useInvoices, useSuppliers, useVaults, useCategories) تُطبّق هذا النمط.
- **الإحصائيات**: تُجلب عبر `useQuery` مع `queryKey` مناسب (مثل `['sales-summaries', companyId, startDate, endDate]`). لا يوجد `unifiedQuery` منفصل — الاستدعاء موحّد عبر الهوكات.
- **الـ API**: `src/services/api.js` — واجهة موحدة؛ الترقيم (`page`, `pageSize`) مدعوم في استدعاءات القوائم.
- **التحميل الكسول**: المسارات تُحمَّل بـ `React.lazy` و `Suspense` لتقليل الحزمة الأولى.
- **المكتبات الثقيلة**: Dynamic Import لـ XLSX و PDF عند الضغط على "تصدير" فقط.
- **مكوّنات موحدة**: `LoadingSpinner` و `ErrorMessage` (مع رموز من docs/ERROR_CODES.md) لمعالجة التحميل والأخطاء في كل الشاشات.

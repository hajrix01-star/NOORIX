# تحسينات حركات الخزينة — Pagination + فلترة زمنية

## ما تم تنفيذه

### 1. الترقيم (Pagination)
- **قبل:** جلب 500 صف ثابت، بدون ترقيم
- **بعد:** 50 صف/صفحة مع أزرار التنقل (‹ › « »)
- إعادة تعيين الصفحة إلى 1 عند تغيير الفترة الزمنية

### 2. فلترة زمنية إلزامية
- **Backend:** يتطلب `startDate` و `endDate` في طلب حركات الخزينة
- **Frontend:** يستخدم الفترة من DateFilterBar (شهر/يوم/نطاق)
- **Fallback:** إذا لم تُمرَّر فترة، يُستخدم الشهر الحالي تلقائياً

### 3. الإجماليات
- **عند الترقيم:** عرض إجمالي الدائن/المدين للفترة الكاملة (من `vault.totalIn` / `vault.totalOut`)
- **عند صفحة واحدة:** إجمالي الصفحة الحالية

### 4. التصدير Excel
- يجلب **كل** الحركات للفترة (حتى 10000) عند الضغط على Excel
- يتضمن صف الإجمالي في نهاية الملف

### 5. حدود الأمان (Backend)
- `pageSize` محدود بين 1 و 10000
- `page` لا يقل عن 1

## قائمة الاختبار

### اختبار أساسي
- [ ] فتح شاشة الخزائن
- [ ] اختيار فترة (شهر/يوم/نطاق)
- [ ] الضغط على خزينة → تظهر حركاتها
- [ ] التحقق من ظهور الترقيم عند وجود أكثر من 50 حركة
- [ ] التنقل بين الصفحات (‹ › « »)
- [ ] تغيير الفترة → العودة للصفحة 1

### اختبار التصدير
- [ ] تصدير Excel → الملف يحتوي كل الحركات للفترة
- [ ] طباعة PDF → تعمل (الصفحة الحالية)

### اختبار الحواف
- [ ] خزينة بدون حركات → رسالة "لا توجد بيانات"
- [ ] فترة بدون حركات → جدول فارغ
- [ ] أكثر من 50 حركة → الترقيم يظهر

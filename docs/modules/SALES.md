# وحدة المبيعات (Sales)

مواصفة وحدة المبيعات في نوركس.

---

## 1. الهدف

- **ليست** لتسجيل المبيعات التفصيلية عند كل عميل (تلك في نقاط البيع).
- **هي** لتسجيل **ملخص نهاية الدوام**: الكاشير يدخل المبيعات في نقاط البيع؛ آخر الدوام يدخل **إجمالي المبيعات** و**قنوات المبيعات** و**عدد العملاء** إلى نوركس ويحفظها.
- إمكانية **تحديد يوم العملية** (مثلاً إدخال مبيعات أمس بتغيير تاريخ العملية فتُحفظ في يوم أمس).
- بعد الحفظ: **رسالة حفظ** ثم **إرسال الملخص إلى واتساب عبر الويب**.

---

## 2. البيانات المُدخلة والمُخزّنة

| الحقل | الوصف |
|--------|--------|
| **تاريخ العملية** | تاريخ يختاره المستخدم (افتراضي اليوم؛ يمكن تغييره لأمس أو أي يوم). يُحفظ كـ Transaction Date. |
| **إجمالي المبيعات** | المبلغ الإجمالي (باستخدام Decimal). |
| **قنوات المبيعات** | تفصيلة حسب القناة (مثلاً: كاش، شبكة) مع مبلغ كل قناة، أو إجمالي فقط. |
| **عدد العملاء** | رقم صحيح (عدد المعاملات أو الفواتير حسب التعريف). |
| **الخزينة/الخزائن** | ربط المبلغ بالخزينة: إما خزينة واحدة للإجمالي، أو توزيع حسب القناة (كاش → خزينة الكاش، شبكة → خزينة البنك). |

---

## 3. القيد المزدوج

- العملية **دخل (Inflow)**:
  - **مدين**: حساب الخزينة (أو خزائن حسب القنوات).
  - **دائن**: حساب إيراد المبيعات.
- إن وُجدت قنوات (كاش / شبكة): إما قيد واحد (خزينة واحدة)، أو قيود/تفصيل بحيث كل قناة مرتبطة بخزينةها.

---

## 4. بعد الحفظ

- عرض **رسالة حفظ ناجح** في الواجهة.
- **إرسال الملخص إلى واتساب عبر الويب** (API أو خدمة واتساب): تاريخ العملية، إجمالي المبيعات، تفصيلة القنوات، عدد العملاء. المستلم وطريقة الربط (رقم، مجموعة، خدمة) تُحدد لاحقاً.

---

## 5. الربط بالمشروع

- المسار: `/sales/new` (أو صفحة مبيعات ملخص نهاية الدوام).
- الـ ID: كل سجل مبيعات له `id`؛ الربط مع الحركات والخزائن بالـ ID فقط.

---

## 6. التصدير والطباعة

- قوائم وسجلات ملخص المبيعات (إن وُجدت شاشة عرض/هستوري) قابلة للتصدير إلى Excel و PDF والطباعة الاحترافية مع اسم الشركة وشعارها وتنسيق احترافي (حسب القواعد العامة في docs/NOORIX_RULES.md §5.1).

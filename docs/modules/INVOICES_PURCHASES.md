# وحدة الفواتير (المشتريات) — Invoices (Purchases)

مواصفة فواتير المشتريات والمصروفات في نوركس.

---

## 1. الفاتورة الفردية

- **البنود (الأصناف) خيارية** وليست إجبارية: يمكن إصدار فاتورة مشتريات **بدون بنود صنف**.
- **إجمالي الفاتورة (المبلغ) إجباري** دائماً.
- باقي الحقول حسب الحاجة: مورد، تاريخ العملية، طريقة الدفع، الخزينة، ضريبة، إلخ.

---

## 2. الإدخال الجماعي (عدة فواتير معاً)

- وجود **زر "إدخال جماعي"** يفتح واجهة على شكل **جدول** يُضاف فيه أكثر من فاتورة في نفس الشاشة.

### 2.1 أعمدة الجدول (كل صف = فاتورة)

| العمود | الوصف |
|--------|--------|
| **رقم الفاتورة** | رقم مرجع الفاتورة (من المورد أو داخلي). |
| **نوع الفاتورة** | مشتريات أو مصروفات. |
| **المورد** | اختيار المورد (بالـ ID). |
| **الفئة** | فئة المشتريات أو المصروفات (مثلاً: مشتريات بضائع، إيجار، رواتب). |
| **الإجمالي شامل الضريبة** | المبلغ الإجمالي شامل الضريبة (إجباري للصف). |
| **(محسوب) المبلغ قبل الضريبة** | يستخرجه النظام من الإجمالي والضريبة. |
| **(محسوب) الضريبة** | يستخرجه النظام. |

### 2.2 تاريخ العملية

- **يوم العملية** واحد للعملية الجماعية (أو قابل للتعديل لكل صف إن لزم لاحقاً).
- إمكانية **تغيير اليوم إلى تاريخ سابق** (مثلاً إدخال فواتير تُسجّل ليوم أمس).

### 2.3 رقم الباتش (Batch) والهستوري

- كل عملية إدخال جماعي لها **رقم باتش واحد** (`batch_id`). كل فاتورة في الجدول تحمل نفس `batch_id`.
- **سجل الإدخالات الجماعية (الهستوري)**: قائمة/شاشة تعرض الباتشات السابقة (رقم الباتش، تاريخ الإدخال، يوم العملية، عدد الفواتير، الإجمالي، المستخدم). من الهستوري يمكن فتح أي باتش لعرض جدول الفواتير أو تعديله أو طباعته أو تصديره.

### 2.4 تعديل الفواتير الجماعي

- عند فتح باتش من الهستوري: يظهر الجدول نفسه مع إمكانية **تعديل** الحقول المسموحة (نوع الفاتورة، الفئة، الإجمالي، المورد، إلخ). عند الحفظ تُحدَّث الفواتير والقيود المرتبطة (قبل الضريبة والضريبة تُعاد حسابهما).

### 2.5 الطباعة والتصدير (إلزامي عند التنفيذ)

- **طباعة الجدول**: زر لطباعة جدول الفواتير في الباتش (من شاشة الباتش أو بعد الإدخال).
- **التصدير**: الجدول (وقوائم الفواتير والجداول الأخرى في الوحدة) **قابلة للتصدير إلى Excel و PDF** مع:
  - **اسم الشركة** و**شعار الشركة** في الترويسة.
  - تنسيق **احترافي منسق وجميل** (ترويسة، حدود، ألوان متناسقة مع ثيم نوركس).
  - **صفحة واحدة** قدر الإمكان (استخدام Scale أو Landscape في PDF؛ إن كثرت الصفوف تُكرَّر الترويسة في كل صفحة).

### 2.6 الحفظ

- عند الحفظ: كل صف في الجدول يُسجّل كحركة/فاتورة مستقلة في القيد المزدوج (صرف: مدين مصروف/مشتريات، دائن خزينة) مع ربط المورد والفئة وتاريخ العملية و`batch_id`.

---

## 3. القيد المزدوج

- فاتورة مشتريات/مصروفات = **صرف (Outflow)**:
  - **مدين**: حساب المصروف أو المشتريات (حسب الفئة).
  - **دائن**: حساب الخزينة (أو حساب المورد إن كان الدفع آجل).
- الربط: `supplier_id`, `vault_id` (أو طريقة الدفع)، `company_id`.

---

## 4. الربط بالمشروع

- المسار: `/invoices` (قائمة وفاتورة فردية وإدخال جماعي وهستوري الباتشات).
- البنود خيارية؛ الإجمالي إجباري. الإدخال الجماعي = جدول بعدة فواتير + رقم باتش + هستوري + تعديل جماعي + طباعة + تصدير Excel/PDF (اسم الشركة، شعار، تنسيق احترافي، صفحة واحدة قدر الإمكان).

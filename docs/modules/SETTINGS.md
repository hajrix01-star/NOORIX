# وحدة الإعدادات (Settings)

مواصفة قسم الإعدادات — إدارة الشركات، الأدوار والصلاحيات، المستخدمون، النسخ الاحتياطي، وربط الذكاء والـ API بشكل آمن.

---

## 1. إدارة الشركات

- **العميل** (مالك الحساب) يدير **الشركات الخاصة به** من قسم الإعدادات.
- **إضافة شركة**: إنشاء شركة جديدة (اسم، بيانات أساسية) وربطها بالعميل.
- **أرشفة شركة**: أرشفة شركة (إخفاء من القائمة النشطة دون حذف البيانات)؛ يمكن استبعادها من الفلاتر الاعتيادية مع الإبقاء على البيانات للمراجعة.
- **حذف شركة**: حذف منطقي (تعطيل/إخفاء) أو حذف مع قيود — حسب السياسة؛ عدم حذف فعلي للبيانات المالية دون إجراءات تدقيق (حسب قوانين المشروع).
- كل شركة لها `id`؛ الربط في كل الجداول عبر `company_id`.

---

## 2. الأدوار والصلاحيات

- **إنشاء فئات المستخدمين (أدوار)** وتعريف **صلاحيات** كل دور على الأقسام والعمليات.
- **صلاحيات نموذجية**:
  - **مالك**: صلاحيات مطلقة كاملة — جميع الشركات، جميع الأقسام (إعدادات، شركات، مستخدمون، مبيعات، فواتير، خزائن، طلبات، تقارير، موارد بشرية، محادثة ذكية، نسخ احتياطي، ذكاء/API).
  - **محاسب**: الاطلاع على **جميع الشركات**؛ أقسام **الفواتير** و**المبيعات** — الاطلاع، الإدخال، المراجعة، والتعديل؛ والتقارير المالية ذات الصلة.
  - **كاشير**: صلاحية **إدخال المبيعات** و**الطلبات** فقط (بدون تعديل فواتير أو إعدادات).
- إمكانية **إنشاء أدوار مخصصة** وربط صلاحيات (قراءة، إدخال، مراجعة، تعديل، حذف) لكل قسم حسب الحاجة.
- **المسمى الوظيفي**: يمكن ربط المستخدم بمسمى وظيفي (للعرض في الشهادات والتقارير) بالإضافة إلى الدور.

## 3. المستخدمون

- **إنشاء مستخدم**: إضافة مستخدم جديد (بيانات الدخول، الاسم، إلخ).
- **ربط المستخدم بالصلاحية**: تعيين **دور** (فئة المستخدم) له؛ يحدد الدور الأقسام والعمليات المسموح بها.
- **ربط بالشركات**: تحديد الشركات التي يمكن للمستخدم الوصول إليها (جميع الشركات للمحاسب/المالك، أو شركات محددة).
- **المسمى الوظيفي**: حقل مسمى وظيفي (مثل "محاسب"، "كاشير") للعرض والربط مع قسم الموارد البشرية إن لزم.

---

## 4. النسخ الاحتياطي (Backup)

- **قسم احترافي مفصل** داخل الإعدادات مخصص للنسخ الاحتياطي.
- **أنماط النسخ**:
  - **نسخ كامل لجميع الشركات**: نسخة واحدة تشمل بيانات كل الشركات التابعة للعميل.
  - **نسخ كل شركة لحالها**: إمكانية إنشاء نسخة احتياطية **لكل شركة على حدة** (اختيار الشركة ثم تنفيذ النسخ).
- **النسخ الدوري (المجدول)**: إمكانية جدولة نسخ احتياطي **دوري** (يومي، أسبوعي، شهري) مع تحديد الوقت؛ التنفيذ تلقائي حسب الجدول.
- **واجهة واضحة**: عرض خيارات النسخ (كامل / حسب الشركة)، الجدولة، وتاريخ آخر نسخة وعدد النسخ المحفوظة.
- **تقرير النسخ — نجاح أو فشل**: بعد كل عملية نسخ (يدوية أو مجدولة) يُسجّل **نتيجة التنفيذ** (نجاح / فشل) مع سبب الفشل إن وجد؛ وعرض **تقرير بالعمليات الأخيرة** (تاريخ، نطاق النسخ، حالة نجاح/فشل، رابط التحميل إن وُجد). التقرير قابل للتصفية والتصدير (Excel/PDF) مع اسم الشركة وشعارها.

---

## 5. الذكاء المستخدم وربط الـ API

- **قسم مخصص** داخل الإعدادات: **الذكاء المستخدم** (مزود الذكاء الاصطناعي للمحادثة الذكية وتحليل كشف الحساب وغيرها).
- **إضافة API بشكل آمن ومحترف ومحمي**:
  - حقل (أو حقول) لإدخال **مفتاح API** أو **رابط النقطة الطرفية** للخدمة المستخدمة؛ لا يُخزَّن المفتاح كنص واضح في الواجهة بعد الحفظ؛ يُستخدم التشفير أو تخزين آمن من جانب الخادم.
  - **عدم عرض المفتاح كاملاً** في الواجهة بعد الحفظ (إظهار جزء منه فقط مع إمكانية "تحديث المفتاح").
  - الصلاحية: **مالك أو مسؤول** فقط يمكنه إضافة/تعديل إعدادات الذكاء والـ API.
- **الربط مع المحادثة الذكية وتحليل الكشف**: بعد إعداد الـ API بشكل صحيح، وحدات "المحادثة الذكية" و"تحليل كشف الحساب" تستخدم هذا الإعداد دون إدخال المفتاح في كل مرة؛ الطلبات تُرسل من الخادم (Backend) لتجنب كشف المفتاح في المتصفح.
- **حماية**: استخدام HTTPS، عدم تسريب المفتاح في السجلات أو الاستجابات الأمامية؛ التحقق من الصلاحيات قبل أي تغيير.

---

## 6. سجل التدقيق (Audit log)

- **العمليات المُسجَّلة**: تغيير صلاحيات أو أدوار، إضافة/أرشفة/حذف شركة، إنشاء أو تعديل مستخدم، تغيير مفتاح API أو إعدادات الذكاء، إقفال شهر محاسبي، تنفيذ نسخ احتياطي.
- **ما يُسجّل**: معرّف المستخدم، التاريخ والوقت، نوع العملية، المعرّف المتأثر (إن وجد)، ملخص التغيير (قبل/بعد إن أمكن). لا تخزين كلمات مرور أو مفاتيح كاملة.
- **الوصول**: المالك أو مسؤول فقط يمكنه عرض سجل التدقيق؛ التصدير حسب الحاجة مع اسم الشركة والشعار.

---

## 7. المسار والتصدير والطباعة (إلزامي عند التنفيذ)

- المسار: `/settings` (تبويبات: شركات، أدوار وصلاحيات، مستخدمون، نسخ احتياطي، الذكاء والـ API، سجل التدقيق).
- تقارير النسخ الاحتياطي (نجاح/فشل) وقوائم الشركات/المستخدمين وسجل التدقيق قابلة للتصدير Excel/PDF والطباعة الاحترافية مع اسم الشركة وشعارها وتنسيق احترافي (حسب docs/NOORIX_RULES.md §5.1).

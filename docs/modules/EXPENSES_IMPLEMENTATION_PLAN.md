# خطة تنفيذ وحدة المصاريف الثابتة والمتغيرة

> خطة احترافية مفصّلة لتنفيذ قسم المصاريف الثابتة والمتغيرة في Noorix — محاذاة مع المعمارية الحالية وقواعد المحاسبة المزدوجة.

---

## 1. نظرة عامة

| البند | التفاصيل |
|------|----------|
| **المسار** | `/expenses` |
| **المواصفة** | [EXPENSES_FIXED_VARIABLE.md](./EXPENSES_FIXED_VARIABLE.md) |
| **الاعتماديات** | Categories, Vaults, Suppliers, FinancialCore (processOutflow) |
| **البيانات الحالية** | Invoice.kind: `fixed_expense` \| `expense` — موجود في الـ Backend |

---

## 2. التعريفات المحاسبية

| النوع | التعريف | أمثلة | kind في Invoice |
|------|---------|-------|-----------------|
| **ثابت** | مبلغ ثابت أو شبه ثابت يتكرر دورياً | إيجار، اشتراكات ثابتة | `fixed_expense` |
| **متغير** | مبلغ يتغير في كل مرة | كهرباء، ماء، هاتف، صيانة | `expense` |

## 6. API Endpoints (موجودة — استخدام)

| Endpoint | الاستخدام |
|----------|-----------|
| `POST /api/v1/invoices` | تسجيل مصروف فردي (kind: fixed_expense \| expense) |
| `POST /api/v1/invoices/batch` | إدخال جماعي |
| `GET /api/v1/invoices?kind=...&startDate=...` | قائمة مصاريف (فلترة) |
| `PATCH /api/v1/invoices/:id` | تعديل مصروف |
| `GET /api/v1/categories?companyId=...` | فئات المصروفات (type=expense) |
| `GET /api/v1/vaults?companyId=...` | الخزائن |
| `GET /api/v1/suppliers?companyId=...` | الموردون |

## 7. القيد المزدوج (لا تغيير)

- **صرف (Outflow)**: مدين = حساب المصروف (حسب الفئة/kind)، دائن = الخزينة.
- **FinancialCore.processOutflow** يطبّق ذلك تلقائياً.
- **KIND_TO_ACCOUNT_CODE**: `fixed_expense` → EXP-003 (إيجار ومرافق)، `expense` → EXP-005 (صيانة وتشغيل).
